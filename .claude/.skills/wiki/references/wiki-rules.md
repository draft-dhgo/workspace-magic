# Wiki 기록 규칙 상세

## 에이전트별 기록 의무

모든 에이전트는 업무를 완료할 때마다 **반드시** 위키에 기록을 남겨야 한다. 이것은 선택이 아닌 필수 사항이다.

## 기록 시점

- 주요 산출물(파일)을 생성/수정한 직후
- 에이전트 작업이 종료되기 직전 (마지막 행동으로 기록)
- 에러로 작업이 중단된 경우에도 진행 상황을 기록

## 순번 결정 프로세스

```
1. Glob으로 wiki/{에이전트명}/*.md 스캔
2. 파일명에서 숫자 추출 (0001.md → 1)
3. 최대값 + 1 = 새 순번
4. 4자리 zero-padding 적용
5. 디렉토리가 없으면 생성 후 0001.md 시작
```

## 기록 내용 가이드라인

### 수행 내용 섹션
- 무엇을 했는지 구체적으로 (모호한 표현 금지)
- 참조한 상위 문서의 어떤 부분을 기반으로 작업했는지
- 작업 단계를 순서대로 나열

### 산출물 섹션
- 생성한 파일: 경로 + 간단한 설명
- 수정한 파일: 경로 + 변경 내용 요약
- 삭제한 파일: 경로 + 삭제 사유

### 주요 결정사항 섹션
- 선택지가 있었던 경우: 선택한 옵션과 이유
- 상위 문서에서 모호했던 부분을 어떻게 해석했는지
- 사용자에게 확인받은 사항

### 이슈 & 참고 섹션
- 발견한 문제점이나 잠재적 리스크
- 후속 작업이 필요한 부분
- 다음 에이전트가 알아야 할 사항

## 금지 사항

- 이전에 작성된 위키 기록을 수정하지 않는다
- 다른 에이전트의 위키 폴더에 기록하지 않는다
- 순번을 건너뛰지 않는다 (0001 → 0003 금지)
- 빈 기록을 남기지 않는다
