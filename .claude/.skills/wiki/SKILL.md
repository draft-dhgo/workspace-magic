# Wiki 스킬

에이전트들이 수행한 업무를 체계적으로 기록하고 관리하는 위키 시스템입니다.

## When to Activate

다음 상황에서 이 스킬을 활성화합니다:
- 사용자가 **위키/wiki 기록**을 요청할 때
- **작업 이력**을 확인하고 싶어할 때
- "위키 보여줘", "기록 확인" 등의 요청

## 디렉토리 구조

```
wiki/
├── prd-consolidate/     # PRD 통합 스킬 기록
│   ├── 0001.md
│   ├── 0002.md
│   └── ...
├── dev-design/          # 개발 설계 에이전트 기록
│   ├── 0001.md
│   ├── 0002.md
│   └── ...
├── test-design/         # 테스트 설계 에이전트 기록
│   ├── 0001.md
│   └── ...
├── implement/           # 구현 에이전트 기록
│   ├── 0001.md
│   └── ...
└── test-write/          # 테스트 작성 에이전트 기록
    ├── 0001.md
    └── ...
```

## 순번 규칙

1. 기록 전 해당 에이전트 폴더를 Glob(`wiki/{에이전트명}/*.md`)으로 스캔
2. 가장 큰 번호를 찾아 +1
3. 폴더가 비어있거나 없으면 `0001.md`부터 시작
4. 4자리 zero-padding (0001, 0002, ..., 9999)

## 기록 템플릿

각 기록 파일은 다음 구조를 따릅니다:

```markdown
# {작업 제목}

> 에이전트: {에이전트명}
> 일시: {YYYY-MM-DD HH:MM}
> 상위 문서: {참조한 상위 문서 경로}

## 수행 내용

{무엇을 했는지 구체적으로 기술}

## 산출물

{생성/수정한 파일 목록}
- `{파일경로}` - {설명}

## 주요 결정사항

{작업 중 내린 중요한 결정과 근거}

## 이슈 & 참고

{발생한 문제, 주의사항, 후속 작업 필요 사항}
```

## 위키 조회

사용자가 위키를 조회할 때:

1. **전체 조회**: `wiki/` 하위 모든 폴더를 스캔하여 에이전트별 기록 수 표시
2. **에이전트별 조회**: 특정 에이전트 폴더의 기록 목록 표시
3. **개별 조회**: 특정 기록 파일 읽기

조회 결과 형식:
```
📖 Wiki 현황

prd-consolidate: 3건 (최근: 0003.md)
dev-design:      2건 (최근: 0002.md)
test-design:     1건 (최근: 0001.md)
implement:       2건 (최근: 0002.md)
test-write:      1건 (최근: 0001.md)
```

## 참고

- 기록 상세 규칙은 `references/wiki-rules.md` 참조
- 모든 에이전트는 작업 완료 시 **반드시** 위키를 기록해야 함
- 위키는 읽기 전용 이력 — 이전 기록을 수정하지 않음
